# 🔌 ESP32-S3 Secure IoT Firmware Agent

This directory contains the embedded C/C++ firmware application for the **Espressif ESP32-S3** microcontroller. Built on **ESP-IDF v5.4.2**, this application maintains Wi-Fi connectivity and implements a highly secure, cost-optimized, and resilient HTTPS Over-the-Air (OTA) update client.

---

## 🚀 Key Features

*   **Dual-Partition OTA:** Uses native Espressif HTTPS OTA APIs (`esp_https_ota`) to stream firmware updates securely. If a boot partition verification fails, the system automatically rolls back to the stable partition, guaranteeing zero bricked hardware.
*   **Plain-Text Version Polling:** Implements a lightweight plain-text version descriptor (`version.txt`). The device polls this tiny payload first; a full firmware download (`firmware.bin`) is only triggered if the version string differs from the local `esp_app_desc_t.version`. This saves tremendous network bandwidth and S3 data fees.
*   **Kconfig Configurable:** Wi-Fi credentials, CDN endpoints, and polling intervals are decoupled from the code and manageable through standard configuration menus.
*   **RGB Status LED:** Drives a NeoPixel LED on GPIO 48 utilizing the RMT driver, showing interactive random color shifts as a running status indicator.

## ⚙️ Configuration Guide (Kconfig)

Instead of editing C source files to change credentials or endpoints, utilize the standard Espressif interactive configuration menu:

```bash
idf.py menuconfig
```

Navigate to **Cloud OTA Configuration** to configure the following parameters:

| Config Macro | Type | Default Value | Description |
| :--- | :---: | :---: | :--- |
| `CONFIG_WIFI_SSID` | `string` | `"Spider"` | Wi-Fi network SSID |
| `CONFIG_WIFI_PASSWORD` | `string` | `"S147@369s"` | Wi-Fi network password |
| `CONFIG_CDN_SERVER_VERSION_URL` | `string` | `"https://.../version.txt"` | CloudFront CDN URL for the version descriptor |
| `CONFIG_CDN_SERVER_FIRMWARE_URL` | `string` | `"https://.../firmware.bin"` | CloudFront CDN URL for the firmware binary |
| `CONFIG_OTA_POLLING_INTERVAL_SEC` | `int` | `30` | Update polling frequency in seconds (default is 30 for testing) |

The settings are dynamically saved into your local `sdkconfig` file (which is git-ignored) and compiled into `#define CONFIG_` macros.

## 🔨 Local Compilation & Deployment

If you have ESP-IDF v5.4.2 installed locally and a device connected:

### 1. Build the Project
```bash
idf.py build
```
This compiles the source code, embeds the TLS root certificate `ota_server_cert.pem` directly into the binary, and produces the ESP32 app partition executable.

### 2. Flash and Monitor
```bash
idf.py -p <PORT> flash monitor
```
*(Replace `<PORT>` with your serial port, e.g. `/dev/cu.usbserial-110` on Mac or `/dev/ttyUSB0` on Linux).*

## 📦 Version Pipeline (`/scripts`)

The GitOps build pipeline uses CMake + a deploy script to package updates:
1. **`CMakeLists.txt`**: Extracts the latest Git tag (`git describe --tags --always`) at configure time, parses the semantic version (e.g. `v0.0.5`), and passes it to `project(test_OTA VERSION x.y.z)`. ESP-IDF embeds this string in `esp_app_desc_t.version` automatically. CMake also writes `build/version.txt` with the same string.
2. **`after_build.sh`**: Post-build orchestrator that:
    * Verifies `build/version.txt` (generated by CMake) and `build/test_OTA.bin`.
    * Automatically queries the AWS **IMDSv2 Instance Metadata service** to discover its hosting AWS region dynamically.
    * Uploads both files directly to the S3 bucket using the server's IAM profile.
